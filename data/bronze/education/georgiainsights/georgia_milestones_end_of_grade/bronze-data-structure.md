# georgia_milestones_end_of_grade - Bronze Data Structure

## Overview

- Topic: georgia_milestones_end_of_grade
- Source: georgiainsights
- Files: 30 files spanning 2015-2025 (no 2020 file - COVID year). Three files per year (state / system / school) except 2015 where state is a single `.xls` and system/school are zipped bundles of per-grade files. Each year publishes a separate file (or zip) for each detail level; beginning 2022 every Excel file carries all six grades (3-8) as separate worksheets within one workbook.
- Unreadable files: none
- Year representation: Year appears only in the filename / title row (`Spring YYYY Georgia Milestones End-of-Grade ...`); there is no `year` column inside the data. "Spring YYYY" refers to the school year ending in that calendar year (e.g., `Spring 2025` = the 2024-2025 school year).
- Filename-to-data year offset: same (filename year = data year; no `year` column to compare against).
- Detail levels: state, district (system), school - each published in a separate file/zip.
- Percentage scale: All `%` columns are 0-100 across every year and every level. `Percent of Enrolled Students Tested` (2021 only) is also 0-100.
- Checksums generated: 2026-05-22

## Source Provenance

- **Source URL**: Georgia Insights (GaDOE) public data downloads — exact page URL not recorded at acquisition time
- **Retrieved**: unknown — predates provenance tracking
- **Method**: manual download (assumed; not recorded at acquisition time)

## Excel Sheet Structure

These files cannot be read with a normal `pl.read_excel()` call. Every sheet is a **two-row pivot header** - the first header row is a forward-fillable super-header naming each subject (e.g., `English Language Arts`, `Mathematics`, `Reading Status`), and the second header row names the metric within each subject (e.g., `Number Tested`, `Mean Scale Score`, `% Beginning Learner`). Row 0 is an ignorable title banner, rows 1-2 are the two header rows, rows 3+ are data, and the sheet ends with one or more blank rows followed by a `^` / `*` footnote describing the Lexile reading-status threshold. The transform must concatenate the two header rows into single composite column names such as `english_language_arts_mean_scale_score`.

All six grades (3-8) are published for every year. The distribution between files and sheets, however, changes across eras - some years ship one file per grade (zipped), others combine all grades into one workbook with one worksheet per grade:

For every era, transform code must:

1. Open each file (unzipping 2015-2021 system/school bundles first).
2. Enumerate the one-or-more grade-level sheets inside.
3. Skip row 0 (title banner), read row 1 and row 2 as the two header rows, and skip the optional footnote / Lexile-legend rows that follow the data.
4. Build composite column names by forward-filling the super-header row and concatenating `super_header + ' - ' + metric_name`.
5. Tag every row with `grade = sheet_grade`, `year = filename_year`, and `detail_level = state | district | school`.
6. Unpivot the resulting wide frame so that `subject` and `metric_name` become key columns and the numeric score becomes a single `value` column (the typical tidy shape used by the gold star schema).

## Summary

Georgia Milestones End-of-Grade (EOG) scores: the results of Georgia's annual state assessment for students in Grades 3-8. The raw files publish, for every subject × grade × location (state / district / school), the **Number Tested**, **Mean Scale Score** (2022+ also **Standard Deviation**), and the percent of test-takers falling into each of four **achievement levels** (Beginning Learner / Developing Learner / Proficient Learner / Distinguished Learner) plus their cumulative variants (% Developing & Above, % Proficient & Above). For ELA specifically the files also publish **Reading Status** metrics - `% Below Grade Level` and `% Grade Level or Above` - using grade-specific Lexile thresholds (520L for grade 3, 740L for grade 4, 830L for grade 5, 925L for grade 6, 970L for grade 7, 1010L for grade 8). Beginning 2018 some subjects split into `EOG`, `EOC`, and `EOG and EOC Combined` variants (middle-schoolers taking end-of-course exams early); 2024+ adds Student Growth Percentile (SGP) blocks for grades 4-8.

## Eras

### Era 1: 2015 (ELA, Math, Science, Social Studies; no Reading Status; no EOC variant)

**State file** (`Spring 2015 EOG All Grades_State.xls`, sheet `State`). Super-header row: `Grade | English Language Arts | Mathematics | Science | Social Studies`. Metric row repeats 8 columns per subject: `Number Tested, Mean Scale Score, % Beginning Learner, % Developing Learner, % Proficient Learner, % Distinguished Learner, % Developing Learner & Above, % Proficient Learner & Above`. 6 data rows (grades 3-8).

**System file** (per-grade `.xls` inside `Spring 2015 EOG System.zip`, 6 files). Super-header: `System Code | System Name | English Language Arts | Mathematics | Science | Social Studies | RESA`. Same 8 metrics per subject. ~195 districts per grade.

**School file** (per-grade `.xls` inside `Spring 2015 EOG School.zip`, 6 files). Super-header: `System Code | School Code | System Name | School Name | English Language Arts | Mathematics | Science | Social Studies | RESA`. Same 8 metrics per subject. ~1,256 schools per grade at Gr 3-5; ~630 per grade at Gr 6-8.

### Era 4: 2018-2019 (adds Reading Status and EOG / EOC / Combined variants)

Two new additions versus earlier eras:

1. A `Reading Status` super-header block that precedes `English Language Arts`, with metrics `Number Tested | % Below Grade Level | % Grade Level or Above` (Lexile thresholds are embedded in the metric header for per-grade sheets; in 2018-2021 state-level workbooks that stack all six grades in one sheet the Lexile cut-points are moved to a separate legend block below the data).
2. For middle-schoolers taking End-of-Course (EOC) tests early, the file publishes three parallel super-header blocks per subject: `English Language Arts - EOG`, `English Language Arts - EOC`, `English Language Arts - EOG and EOC Combined` - and the same trio for Mathematics, Science, and Social Studies. Grade 3-5 files still have all `--` in the EOC and Combined blocks because no elementary students take EOCs; the columns exist but are fully suppressed.

Row volumes very similar to Era 2 (~1,220 system rows / 6 grades, ~5,680 school rows total). State file has 6 data rows plus an appended Lexile-threshold legend that transform code must drop.

### Era 5: 2021 (COVID-era participation; no 2020 file)

No EOG file was published for 2020 (the pandemic cancelled administration). The 2021 file retains the Era-4 schema but **adds a `Percent of Enrolled Students Tested` column in every subject super-block** (testing was optional in 2021, so participation rates are essential context). Reading Status also gains this column. Everything else - subject coverage, suppression marker (`--`), row layout - matches 2019. ~1,250 system rows / 6 grades, ~5,600 school rows total. State: 6 rows + Lexile legend.

Grade 8 2021 system files have a merged-cell artifact: the ELA-EOG super-header spans over two columns, so the second header row reads `Number Tested, (blank), Mean Scale Score, % Beginning Learner, ...` - the transform must tolerate an extra blank column inside the ELA block, or simply drop columns whose metric name is blank.

#### Suppression markers

| Column | Non-numeric values observed |
|--------|-----------------------------|
| Every `%`, score, and Number-Tested column where publication thresholds not met | `--` (1,900+ across all 2021 system files) |

### Era 6: 2022 (workbook consolidated, no more EOC split, adds Standard Deviation)

Starting 2022 the three levels each ship a single workbook containing six worksheets (one per grade: `State - Grade 3`, ..., `School - Grade 8`). The per-grade zipping disappears. Schema changes:

- Subject super-headers simplify to just `Reading Status`, `English Language Arts`, `Mathematics`. The `EOG / EOC / Combined` triples are gone (EOC moves to `georgia_milestones_end_of_course`).
- Each subject block gains a `Standard Deviation` metric (so 9 metrics per subject instead of 8).
- Grade 6-8 system and school files drop the `Science - EOC` / `Social Studies - EOC` / `Combined` blocks from prior eras; 2022 has no Science or Social Studies block at all for grade 6-8 (those subjects return in 2023).
- `Percent of Enrolled Students Tested` from 2021 is not included.

Row volumes: state 6 rows total (1 per grade sheet × 6 sheets); system ~1,260 rows total; school ~5,600 rows total.

## ETL Considerations

- **Header variant (verified 2026-06-10):** grade 5 State sheets in both the 2024 and 2025 workbooks contain a `% SGPTypical Growth` header (missing space) alongside the normal `% SGP Typical Growth` on other sheets — the transform's header normalization must tolerate both spellings.

- **Multi-row pivot header.** Every sheet has a banner row (row 0), a super-header row (row 1) naming subjects, and a metric row (row 2). Polars' `read_excel` will fold these into unusable names like `__UNNAMED__1`. The transform must read the first three rows explicitly (via openpyxl for `.xlsx` or xlrd for `.xls`), forward-fill the super-header row, and concatenate `subject + '_' + metric` to build working column names before parsing data rows.
- **Unzipping 2015-2021 system/school bundles.** 2015-2021 ship system and school files as `.zip` archives of six per-grade Excel files. The 2015 zips additionally double-nest the content inside a redundant outer directory (`Spring 2015 EOG - School Level/Spring 2015 EOG - School Level/Gr3_School.xls`) - when unzipping, flatten directory paths (e.g., use `unzip -j` or Python's `zipfile` with `os.path.basename`).
- **2015 `.xls` format.** The 2015 files are legacy `.xls` (BIFF format) and cannot be opened by `openpyxl`. Use `xlrd` or Polars' calamine backend with `.xls` support. School codes in 2015 are zero-padded strings (`'0177'`), but in 2016+ they sometimes appear as integers (`177`) - cast to zero-padded 4-character strings during transform so that downstream joins on `school_code` match the dimension table.
- **Title-cased 2016 system names.** Every year except 2016 stores `System Name` in ALL CAPS. 2016 uses Title Case (`Appling County` vs `APPLING COUNTY`). Normalize to Title Case at the gold layer (or ALL CAPS - whatever matches the districts dimension) before joining.
- **Footnote / legend rows at the bottom of each sheet.** Most sheets end with a blank row and then a footnote starting with `^` or `*` that describes the Lexile threshold. The 2018-2021 state files stack a multi-row Lexile-threshold legend beneath the data (rows 13+). Drop any row whose first cell is a long string (>50 chars) or starts with `^` / `*`, and drop rows that are entirely blank.
- **Suppression marker.** The universal suppression marker is the literal string `--` (two dashes). It appears in every numeric column whenever the cell population threshold is not met. There are no other suppression markers (no `*`, `TFS`, `N/A`, or `No Data` as used in other GaDOE topics). Convert `--` -> null before casting numerics.
- **Grade-specific Lexile thresholds.** The `% Below Grade Level` / `% Grade Level or Above` cut-points are grade-specific: 520L for Grade 3, 740L for Grade 4, 830L for Grade 5, 925L for Grade 6, 970L for Grade 7, 1010L for Grade 8. Gold schema should carry a single pair of columns (e.g., `reading_pct_below_grade_level`, `reading_pct_grade_level_or_above`); the threshold metadata should travel in documentation rather than in column names so that the gold schema is grade-agnostic.
- **EOG / EOC / Combined columns in 2018-2021 elementary grades.** The `EOC` and `EOG and EOC Combined` super-blocks exist for all grades in 2018-2021 but are fully `--` for Grades 3-5 (no elementary EOC testers). Transform can safely drop these blocks for elementary grades; for Grades 6-8 they should be kept and represented as variant subjects (or filtered out if the gold spec is EOG-only).
- **2017 Science and Social Studies missing.** 2017 is missing Science and Social Studies entirely. The gold fact table will have null/absent values for those subjects in the 2017 slice - this is expected, not a data error.
- **2021 merged-cell artifact (Grade 8 system).** The 2021 Grade 8 system file has a merged super-header cell that spreads over two columns within the ELA block, so one column in the metric header row comes out blank. Either drop columns where the metric name is blank, or test for the extra empty cell and compress it out.
- **Three-level partitioning.** State, district, and school-level data live in three separate files per year, not in different rows of the same file. Transform should read all three and tag each row with a `detail_level` column ('state' / 'district' / 'school') before concatenation. There is NO "All" / state-total row interleaved in the system or school files.
- **No demographic subgroup breakdown.** Unlike CCRPI Content Mastery or Attendance Dashboard, these EOG files publish aggregate results only - there is no `SUBGROUP_DESC` or `Reporting Category` column. The gold schema should either omit `demographic` (and rely on `all_students` being implicit) or hard-code `demographic = 'All Students'` to stay consistent with the star schema's demographic foreign key.
- **No year column in the data.** Year must be extracted from the filename / title row. Every grade/subject/location/year combination is unique without a year column, but the transform still needs to stamp `year` onto every row.
- **Subject-block column count varies by year/grade.** Not every file has the same subjects (2017 omits Science/SS; 2022 omits Science/SS for Gr 6-8; Gr 8 in 2023-2025 adds HS Physical Science; 2024-2025 Gr 4-8 adds SGP). Transform should handle a dynamic list of `(subject, metric)` tuples per sheet rather than assuming a fixed column layout.
- **Tidy unpivot before export.** The gold fact table should be fully melted: one row per `year × detail_level × system_code × school_code × grade × subject × metric_name = value`. This keeps the schema stable across the evolving set of subjects and avoids ever-changing wide column counts in the fact table.

- **2026-06-12 (transform authoring)** — claims re-verified against all 30 bronze files during the clean-start rebuild; evidence is from raw `header=None, dtype=str` / `header=[1,2], dtype=str` reads of the named files/sheets.
  - **The EOG / EOC / Combined split starts in 2016, not 2018 (Era 4 claim), and never applies to Social Studies.** `Spring_2016_EOG-State_Level.xlsx` and the 2016 grade 6-8 System/School members already carry `Mathematics - EOG / EOC / EOG and EOC Combined` and (grade 8) `Science - EOG / EOC / EOG and EOC Combined` super-headers with real data.   - **Era 3 (2017) is NOT "ELA and Mathematics only".** Per-grade members: grades 3-4 are ELA+Math only; grade 5 carries `Science` and `Social Studies` (plain); grades 6-7 carry `Mathematics - EOG / EOC / Combined`; grade 8 carries the Mathematics and Science trios plus plain `Social Studies` (e.g. `Spring_2017_EOG-System_Level.zip :: … Grade 8` row 1). The 2017 state file likewise carries the Math/Science EOC and Combined columns.
  - **Per-grade 2018-2019 files carry EOC blocks only at grades 6-8.** The Era 4 claim that "Grade 3-5 files still have all `--` in the EOC and Combined blocks; the columns exist but are fully suppressed" is true only of the all-grades STATE files (where grade 3-5 rows are `--` under those columns).
